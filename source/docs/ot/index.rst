OT/ICS security
====================================

.. image:: /_static/images/ot-company-location.png
   :alt: UUPL

Industrial control systems were designed to be reliable. Security came later, often much later, and the gap between
the two is where most of the risk lives.

Assembly, C, and C++ remain the languages of PLCs, RTUs, embedded firmware, and the engineering workstations that
configure them. The patterns that cause vulnerabilities in those languages predate the internet and have not gone away.
The challenge is applying controls that work within real-time and resource constraints that web application security
guidance assumes away.
